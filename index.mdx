Architectury API is an intermediary library for Minecraft mod development. It provides a unified API surface that abstracts the differences between [Fabric API](https://github.com/FabricMC/fabric) and [Forge API](https://github.com/MinecraftForge/MinecraftForge), letting you write your mod logic once and run it on both platforms.

## Get started in minutes

<Steps>
  <Step title="Add the dependency">
    Add Architectury API to your `common`, `fabric`, and `forge` subprojects using Gradle.
    See [Installation](/getting-started/installation) for exact dependency coordinates.
  </Step>
  <Step title="Register your content">
    Use `DeferredRegister` to register blocks, items, and other game objects in a platform-agnostic way.
    See [Registering Content](/guides/registering-content).
  </Step>
  <Step title="Subscribe to events">
    Register listeners on Architectury events like `PlayerEvent.PLAYER_JOIN` or `LifecycleEvent.SETUP`.
    See [Events Guide](/guides/events).
  </Step>
  <Step title="Use @ExpectPlatform for platform-specific code">
    When you need loader-specific behavior, use the `@ExpectPlatform` annotation to provide separate implementations per platform.
    See [ExpectPlatform](/concepts/expect-platform).
  </Step>
</Steps>

<Note>
  Architectury API is only one part of the Architectury ecosystem. You also need [Architectury Loom](https://github.com/architectury/architectury-loom) and the [Architectury Plugin](https://github.com/architectury/architectury-plugin) for your Gradle build. See [Project Setup](/getting-started/project-setup) for the complete picture.
</Note>
