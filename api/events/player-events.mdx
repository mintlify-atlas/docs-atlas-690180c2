All events listed here are server-side unless otherwise noted. The listener type and method signature are extracted from the source interfaces.

## Event reference

<AccordionGroup>
  <Accordion title="PLAYER_JOIN — Event&lt;PlayerEvent.PlayerJoin&gt;">
    Invoked after a player joins a server level.

    **Listener interface:** `PlayerEvent.PlayerJoin`

    ```java
    void join(ServerPlayer player);
    ```

    | Parameter | Type | Description |
    |---|---|---|
    | `player` | `ServerPlayer` | The player who joined. |

    **Forge equivalent:** `PlayerLoggedInEvent`
  </Accordion>

  <Accordion title="PLAYER_QUIT — Event&lt;PlayerEvent.PlayerQuit&gt;">

  <Accordion title="PLAYER_RESPAWN — Event&lt;PlayerEvent.PlayerRespawn&gt;">
    Invoked when a player is respawned (e.g., after dying or leaving the End). To copy data to the new player object, use `PLAYER_CLONE` instead.

    **Listener interface:** `PlayerEvent.PlayerRespawn`

    ```java
    void respawn(ServerPlayer newPlayer, boolean conqueredEnd);
    ```

    | Parameter | Type | Description |
    |---|---|---|
    | `newPlayer` | `ServerPlayer` | The newly created player instance. |
    | `conqueredEnd` | `boolean` | `true` when the player completed the End and is returning to the overworld. |

    **Forge equivalent:** `PlayerRespawnEvent`
  </Accordion>

  <Accordion title="PLAYER_CLONE — Event&lt;PlayerEvent.PlayerClone&gt;">
    Invoked when the player object is recreated at respawn. Use this to transfer custom data from the old player to the new one.

    **Listener interface:** `PlayerEvent.PlayerClone`

    ```java
    void clone(ServerPlayer oldPlayer, ServerPlayer newPlayer, boolean wonGame);
    ```

    | Parameter | Type | Description |
    |---|---|---|
    | `oldPlayer` | `ServerPlayer` | The player instance before respawn. |
    | `newPlayer` | `ServerPlayer` | The player instance after respawn. |
    | `wonGame` | `boolean` | `true` when the player completed the End. |

    **Forge equivalent:** `PlayerEvent.Clone`
  </Accordion>

    **Listener interface:** `PlayerEvent.FillBucket`

    ```java

        Level level,
        ItemStack stack,
        @Nullable HitResult target

    ```

    | Parameter | Type | Description |
    |---|---|---|
    | `player` | `Player` | The player filling the bucket. |
    | `level` | `Level` | The level the player is in. |

    | `target` | `@Nullable HitResult` | The ray-cast target, or `null`. |

    **Returns:** `CompoundEventResult<ItemStack>` — return `CompoundEventResult.interruptTrue(filledStack)` to provide a replacement result.

## Usage examples

### Greeting players on join

```java
PlayerEvent.PLAYER_JOIN.register(player -> {
    player.sendSystemMessage(
        Component.literal("Welcome, " + player.getGameProfile().getName() + "!")
    );
});
```

### Cancelling item drops

```java
PlayerEvent.DROP_ITEM.register((player, entity) -> {
    ItemStack stack = entity.getItem();
    if (stack.is(Items.DIAMOND)) {
        // Prevent the player from dropping diamonds
        return EventResult.interruptFalse();
    }
    return EventResult.pass();
});
```

### Copying custom data on respawn

```java
PlayerEvent.PLAYER_CLONE.register((oldPlayer, newPlayer, wonGame) -> {
    CompoundTag oldData = oldPlayer.getPersistentData();
    CompoundTag newData = newPlayer.getPersistentData();
    if (oldData.contains("MyMod:customValue")) {
        newData.put("MyMod:customValue", oldData.get("MyMod:customValue"));
    }
});
```
