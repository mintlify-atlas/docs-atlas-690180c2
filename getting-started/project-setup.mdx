An Architectury project is a Gradle multi-project build with three subprojects: `common`, `fabric`, and `forge`. Shared logic lives in `common`. Each platform subproject compiles the common code against its loader and produces a loader-specific jar.

## Project structure

```
my-mod/
├── settings.gradle          # declares subprojects
├── build.gradle             # root build config, plugin versions
├── gradle.properties        # shared version properties
├── common/
│   ├── build.gradle
│   └── src/main/java/       # shared mod code
├── fabric/
│   ├── build.gradle
│   └── src/main/java/       # Fabric-specific code and impls
└── forge/
    ├── build.gradle
    └── src/main/java/       # Forge-specific code and impls
```

The `common` subproject never depends on a specific loader. It compiles against Minecraft mappings and the Architectury API common artifact only. Both platform subprojects depend on `common` and include its compiled output in their final jars via a `shadowCommon` configuration.

## Set up the subprojects

<Steps>
  <Step title="Declare subprojects in settings.gradle">
    Register all three subprojects so Gradle resolves them as a multi-project build.

    ```groovy settings.gradle
    pluginManagement {
        repositories {
            maven { url "https://maven.fabricmc.net/" }
            maven { url "https://maven.architectury.dev/" }
            maven { url "https://files.minecraftforge.net/maven/" }
            gradlePluginPortal()
        }
    }

    include("common")
    include("fabric")
    include("forge")

    rootProject.name = "my-mod"
    ```
  </Step>

  <Step title="Apply plugins in the root build.gradle">
    Apply `architectury-plugin` at the root level and `dev.architectury.loom` to subprojects. Set the target Minecraft version once here so all subprojects share it.

    ```groovy build.gradle
    plugins {
        id "architectury-plugin" version "3.4-SNAPSHOT"
        id "dev.architectury.loom" version "1.0-SNAPSHOT" apply false
    }

    architectury {
        minecraft = project.minecraft_version
    }

    subprojects {
        apply plugin: "dev.architectury.loom"

        loom {
            silentMojangMappingsLicense()
        }

        dependencies {
            minecraft "com.mojang:minecraft:${rootProject.architectury.minecraft}"
            mappings loom.officialMojangMappings()
        }
    }

    allprojects {
        apply plugin: "java"
        apply plugin: "architectury-plugin"

        group = project.maven_group
        version = project.mod_version
    }
    ```
  </Step>

  <Step title="Configure the common subproject">
    In `common/build.gradle`, tell Architectury Plugin which platforms this common code targets. This generates the transformation stubs `@ExpectPlatform` needs.

    ```groovy common/build.gradle
    architectury {
        common(["fabric", "forge"])
    }

    dependencies {
        // Use fabric-loader for @Environment annotations only.
        // Do NOT use other Fabric Loader classes in common code.
        modImplementation "net.fabricmc:fabric-loader:${rootProject.fabric_loader_version}"

        modCompileOnly "dev.architectury:architectury:6.6.92"
    }
    ```
  </Step>

  <Step title="Configure the Fabric subproject">
    In `fabric/build.gradle`, tell Architectury Plugin this is the Fabric platform. Include the common compiled output via `shadowCommon` so it is bundled into the final Fabric jar.

    ```groovy fabric/build.gradle
    architectury {
        platformSetupLoomIde()
        fabric()
    }

    configurations {
        common
        shadowCommon
        compileClasspath.extendsFrom common
        runtimeClasspath.extendsFrom common
        developmentFabric.extendsFrom common
    }

    dependencies {
        modImplementation "net.fabricmc:fabric-loader:${rootProject.fabric_loader_version}"
        modImplementation "net.fabricmc.fabric-api:fabric-api:${rootProject.fabric_api_version}"

        modImplementation "dev.architectury:architectury-fabric:6.6.92"

        common(project(path: ":common", configuration: "namedElements")) { transitive false }
        shadowCommon(project(path: ":common", configuration: "transformProductionFabric")) { transitive false }
    }
    ```
  </Step>

  <Step title="Configure the Forge subproject">
    In `forge/build.gradle`, apply the Forge platform configuration. Include any Mixin configs your common code uses.

    ```groovy forge/build.gradle
    architectury {
        platformSetupLoomIde()
        forge()
    }

    loom {
        forge {
            mixinConfig "my-mod.mixins.json"
        }
    }

    configurations {
        common
        shadowCommon
        compileClasspath.extendsFrom common
        runtimeClasspath.extendsFrom common
        developmentForge.extendsFrom common
    }

    dependencies {
        forge "net.minecraftforge:forge:${rootProject.minecraft_version}-${rootProject.forge_version}"

        forgeRuntimeLibrary "dev.architectury:architectury-forge:6.6.92"

        common(project(path: ":common", configuration: "namedElements")) { transitive false }
        shadowCommon(project(path: ":common", configuration: "transformProductionForge")) { transitive false }
    }
    ```
  </Step>

  <Step title="Add entrypoints">
    Each platform needs an entrypoint that initializes your common mod code.

    **Fabric** — register a `main` entrypoint in `fabric.mod.json`:

    ```json fabric/src/main/resources/fabric.mod.json
    {
      "schemaVersion": 1,
      "id": "my-mod",
      "version": "${version}",
      "entrypoints": {
        "main": ["com.example.fabric.MyModFabric"]
      },
      "depends": {
        "minecraft": "~1.19.2",
        "fabricloader": ">=0.14.0",
        "fabric-api": ">=0.66.0",
        "architectury": ">=6.6.0"
      }
    }
    ```

    The Fabric entrypoint class calls your common `initialize()` method and registers the mod event bus with Architectury:

    ```java fabric/src/main/java/com/example/fabric/MyModFabric.java
    import net.fabricmc.api.ModInitializer;
    import com.example.MyMod;

    public class MyModFabric implements ModInitializer {
        @Override
        public void onInitialize() {
            MyMod.initialize();
        }
    }
    ```

    **Forge** — use the `@Mod` annotation. On Forge you must also register your mod's event bus with `EventBuses` before calling your common initializer:

    ```java forge/src/main/java/com/example/forge/MyModForge.java
    import dev.architectury.platform.forge.EventBuses;
    import net.minecraftforge.fml.common.Mod;
    import net.minecraftforge.fml.javafmlmod.FMLJavaModLoadingContext;
    import com.example.MyMod;

    @Mod(MyMod.MOD_ID)
    public class MyModForge {
        public MyModForge() {
            EventBuses.registerModEventBus(
                MyMod.MOD_ID,
                FMLJavaModLoadingContext.get().getModEventBus()
            );
            MyMod.initialize();
        }
    }
    ```

    <Warning>
      On Forge, call `EventBuses.registerModEventBus(...)` **before** any Architectury event registrations or registry operations. Calling it after will cause those registrations to be missed.
    </Warning>
  </Step>
</Steps>

## Where to put your code

| Code type | Location |
|---|---|
| Shared game logic, events, registrations | `common/src/main/java/` |
| Fabric-only code (Fabric API calls, quilt compat) | `fabric/src/main/java/` |
| Forge-only code (Forge event hooks, `@Mod` class) | `forge/src/main/java/` |
| `@ExpectPlatform` implementations | `fabric/.../YourClassImpl.java` and `forge/.../YourClassImpl.java` |

## Using `@ExpectPlatform` for platform-specific branches

When your common code needs to do something differently on each loader, declare a `static` method annotated with `@ExpectPlatform` in the common module:

```java common/src/main/java/com/example/platform/PlatformHelper.java
import dev.architectury.injectables.annotations.ExpectPlatform;

public class PlatformHelper {
    @ExpectPlatform
    public static String getLoaderName() {
        throw new AssertionError();
    }
}
```

Then provide the implementation in each platform subproject in the same package, appending `Impl` to the class name:

```java fabric/src/main/java/com/example/platform/PlatformHelperImpl.java
public class PlatformHelperImpl {
    public static String getLoaderName() {
        return "Fabric";
    }
}
```

```java forge/src/main/java/com/example/platform/PlatformHelperImpl.java
public class PlatformHelperImpl {
    public static String getLoaderName() {
        return "Forge";
    }
}
```

See [ExpectPlatform](/concepts/expect-platform) for the full reference, including naming rules and caveats.
